# Консольный Pac-Man (минимальная версия)

Маленькая игра в стиле Pac-Man, написанная на C для консоли Windows.  
Соберите три монетки, уворачиваясь от привидений, в лабиринте 8×8.

## Возможности

- Лабиринт 8×8
- Игрок `a` (Пакман) управляется стрелками
- Четыре привидения: `b` (Блинки), `c` (Пинки), `d` (Инки), `e` (Клайд)
- Собираемые монетки `.` - нужно собрать три для победы
- Случайное движение привидений
- Звуковая обратная связь через `Beep()`
- Цветной вывод с помощью ANSI-управляющих последовательностей
- Частота кадров примерно 3.7 кадра в секунду

## Требования

- Операционная система Windows
- Компилятор C (MSVC, MinGW‑w64 и т.п.)
- Консоль с поддержкой ANSI-последовательностей

## Компиляция

С помощью **MinGW‑w64** (gcc):
```bash
gcc -o pacman.exe pacman.c
```

С помощью **MSVC** (командная строка разработчика):
```bash
cl pacman.c
```

Дополнительные библиотеки не требуются – игра использует только kernel32 и стандартные заголовочные файлы.

## Как играть

Запустите исполняемый файл. Игра начинается сразу.

- **Стрелки** - движение Пакмана (вверх, вниз, влево, вправо)
- Соберите белую точку `.` - каждая монетка увеличивает счёт
- Наберите счёт **3** для победы
- Если любое привидение коснётся Пакмана, игра заканчивается

После завершения выводится сообщение, и программа закрывается.

## Игровая механика

### Карта
Лабиринт хранится в виде 64-битного целого числа, интерпретируемого как 8 строк по 8 бит.  
Бит `1` означает стену (`#`), бит `0` – пустой коридор (` `).  
Начальный вид:
```
########
#      #
# #    #
# #    #
# ##   #
#      #
#      #
########
```

### Игрок (Пакман)
- Начальная позиция: (1,1) (столбец, строка)
- За кадр перемещается на одну клетку в выбранном направлении, *если клетка впереди не стена*
- При каждом успешном шаге звучит короткий сигнал:
  - Вверх: 800 Гц
  - Вниз: 400 Гц
  - Влево: 500 Гц
  - Вправо: 600 Гц

### Привидения
- Каждое привидение (b, c, d, e) начинает в фиксированной точке:
  - Блинки: (6,1)
  - Пинки:  (2,3)
  - Инки:   (1,6)
  - Клайд:  (6,6)
- Каждый кадр привидение пытается двигаться в случайном направлении.
  Порядок попыток: вниз -> вверх -> влево -> вправо.
  Движение в каком-либо направлении происходит, если путь свободен **и** случайное число (0‑3) совпадает с константой направления.
- Привидения могут накладываться друг на друга и не используют алгоритмы поиска пути.
- Столкновение с Пакманом мгновенно завершает игру.

### Монетки
- На карте всегда присутствует одна монетка.
- Она появляется в случайной свободной клетке (координаты 1‑6 по обеим осям) и отображается как `.`.
- Когда Пакман встаёт на монетку, счёт увеличивается, и тут же генерируется новая монетка в допустимой пустой клетке.
- Соберите 3 монетки для победы.

### Счёт
Счёт хранится в двух младших битах переменной состояния монетки.  
После третьей монетки объявляется победа.

## Управление

| Клавиша    | Действие    |
|------------|-------------|
| Стрелка вверх    | Движение вверх    |
| Стрелка вниз     | Движение вниз     |
| Стрелка влево    | Движение влево    |
| Стрелка вправо   | Движение вправо   |

Остальные клавиши игнорируются.

## Примечания

- Игра работает в бесконечном цикле до наступления условия победы или поражения.
- Для создания плавной анимации каждый кадр консоль очищается и курсор возвращается в начало.
- В исходном коде для хранения всех координат и направлений используется упаковка в одно 32-битное целое – намеренно компактный дизайн.
- Если вы не видите цветов или отображаются странные символы, убедитесь, что терминал поддерживает ANSI-последовательности.
